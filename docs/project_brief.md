# RAGBench-CN Project Brief

## 项目是什么

RAGBench-CN 是一个轻量中文 RAG 评测工具。

它不是聊天机器人，也不是新的 RAG 平台。它的目标是用一套中文问题集自动测试知识库问答系统，输出关键词召回率、引用命中率、平均延迟和失败类型报告。

## 为什么做

很多 RAG demo 能回答问题，但缺少评估：

- 不知道是否引用了正确文档。
- 不知道答案是否覆盖关键点。
- 不知道 top-k、chunk size、rerank 调整后有没有变好。
- 面试时很难讲清失败 case 和优化依据。

RAGBench-CN 关注的是“RAG 系统到底准不准”，不是只做一个看起来能回答的 demo。

## 目标用户

- 学生和实习候选人。
- 正在搭 RAGFlow、Dify、FastGPT 或 LangChain 知识库的人。
- 想快速评估中文 RAG 项目效果的人。

## 第一版边界

第一版只做命令行工具：

- 读取中文问题集。
- 调用 mock 或 RAG 系统接口。
- 计算基础指标。
- 输出 Markdown 报告。

暂时不做：

- Web UI。
- 完整 RAG 平台。
- 复杂 LLM-as-judge。
- 多平台深度适配。

## 简历表达

设计并实现轻量中文 RAG 评测工具，支持批量问题集测试、引用命中率、关键词召回率、平均延迟和失败类型统计；基于 RAGFlow 验证知识库问答链路，并输出 Markdown 评估报告和失败 case 分析。
